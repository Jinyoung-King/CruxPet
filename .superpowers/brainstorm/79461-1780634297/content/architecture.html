CruxPet 아키텍처 — 어떻게 연결할까요?

Git hook → 앱 → 펫 상태. 중간 연결 방식 3가지

A

파일 기반 (추천)

Git hook이 ~/.cruxpet/events.json에 이벤트를 기록. 앱이 파일을 감시하다가 읽음. 포모도로 완료도 같은 파일에 기록.

장점

  • 단순하고 안정적
  • 디버깅 쉬움
  • 앱 꺼져 있어도 이벤트 누락 없음

단점

  • 반응 약간 느림 (~1초)
B

macOS 알림 기반

Git hook이 CFNotificationCenter (Darwin 알림)으로 앱에 직접 신호. 즉각 반응.

장점

  • 실시간 반응 (즉각)
  • 파일 I/O 없음

단점

  • 앱 꺼져 있으면 이벤트 유실
  • 구현 복잡
C

SQLite 이벤트 스토어

모든 이벤트를 SQLite DB에 저장. 앱이 주기적으로 쿼리해서 펫 상태 계산. 활동 히스토리·통계 추적 가능.

장점

  • 히스토리 / 통계 가능
  • 이벤트 누락 없음

단점

  • 지금 단계엔 과한 구조
  • 구현 시간 더 필요
